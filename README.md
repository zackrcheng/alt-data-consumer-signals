# DoorDash (DASH) Investment Thesis

A fundamental L/S equity research project that builds a quarterly forecast
model for DASH US Marketplace GOV (Gross Order Volume) using alt-data signals,
quantifies the GOV → revenue → EBITDA (→ CAR) transmission chain, and
generates a long-DASH view.

**Pre-registered Q1 2026 forecast** (saved before the May 6 2026 print):

| | Value | 80% CI |
|---|---|---|
| GOV surprise | **+2.26pp** | [+1.35, +3.94] |
| Implied actual GOV YoY | ~+38.9% | — |
| FactSet consensus YoY | +36.6% | — |
| Cross-sectional spread vs UBER | **+1.59pp** | [+0.25, +4.83] |
| Expected CAR[-1, +2] | +1.0% | [+0.4, +2.1] (conditional mean) |

The pre-registration record is in `outputs/tables/q1_2026_preregistered.csv`,
timestamped 2026-05-05 (one day before the earnings print) and committed to git.

---

## What this repo does

**Question**: Is the market's implied view of DASH's US Marketplace GOV
growth trajectory wrong, and in which direction?

**Approach**: build a multi-pillar thesis stack (Volume / Monetization /
Profitability) anchored in three statistical workflows:

1. **GOV-surprise forecast model** (`src/model_gov.py`) — 48 model variants
   compared on walk-forward (Q1 2023 → Q4 2025). Best variant (LASSO on
   standardized surprise) selected by directional_acc with RMSE tiebreak;
   80% CI from quantile regression. Pre-registered to
   `outputs/tables/q1_2026_preregistered.csv`.

2. **Transmission chain** (`src/model_transmission.py`) — pillar-aligned β
   regressions: GOV → revenue (β1) → contribution margin (β_U) → EBITDA
   margin (β_C). Plus cross-sectional spread DASH − UBER from a parallel
   UBER GB-surprise model (`src/model_crosssectional.py`) for comparison only, not for L/S view.

3. **Event study** (`src/event_study.py`) — β3: CAR[-1, +2] ~ GOV surprise.
   CRSP CIZ daily returns + DSI value-weighted market via WRDS, with
   yfinance/SPY fallback for events past CRSP coverage.

## Reproduce

### Prerequisites

- Python 3.12, environment named `quant` (or any env with the dependencies)
- Dependencies: pandas, numpy, statsmodels, scikit-learn, scipy, seaborn,
  matplotlib, wrds, yfinance, dotenv, pytrends, vaderSentiment
- WRDS subscription (Compustat / IBES / CRSP CIZ) for the financial pulls
- Set `WRDS_USERNAME` in `.env` (copy from `.env.template`); password via
  `~/.pgpass` for non-interactive runs

### Pipeline

```bash
# Data pulls (one-time; can take ~30 min total via WRDS)
python -m src.pull_trends
python -m src.pull_fred
python -m src.pull_prices
python -m src.pull_appstore
python -m src.pull_reddit
python -m src.pull_weather
python -m src.pull_job_postings
python -m src.pull_wrds_compustat
python -m src.pull_wrds_ibes
python -m src.pull_crsp                # CRSP CIZ daily + DSI

# Derived data
python -m src.build_gov_table          # FactSet xlsx → dash/uber_gov_master.csv
python -m src.build_master_df          # all sources → master_df.csv

# Modeling (each script saves to outputs/tables/ + outputs/figures/)
python -m src.model_gov                # 48-variant comparison + pre-reg
python -m src.model_transmission       # β1, β_M, β_U, β_C, β2, chain to Q1 2026
python -m src.model_crosssectional     # UBER + DASH−UBER spread
python -m src.event_study              # β3 + Q1 2026 expected CAR

# Notebooks (human review — re-execute after rerunning model scripts)
jupyter nbconvert --to notebook --execute --inplace notebooks/*.ipynb
```

## Headline findings

The four-pillar L/S note from `notebooks/04_CrossSectional.ipynb` §6:

| Pillar | Reading |
|---|---|
| **Volume** | DASH alt-data → +2.26pp surprise (high directional conviction); UBER → +0.67pp |
| **Monetization** | β1 = +2.59 (p<0.01); revenue surprise +6.5pp implied |
| **Profitability** | β_C = +3.00 (p<0.001); EBITDA margin lift +3.9pp YoY anchored in secular contribution-margin uptrend |
| **Relative** | DASH − UBER spread = +1.59pp, conservative 80% CI [+0.25, +4.83] (lower bound positive) |

**Recommendation**: Long DASH · 3–6 month horizon · small magnitude · medium-high conviction on direction, low-medium conviction on magnitude.

**Key empirical findings that diverged from priors**:

1. The surprise target (`gov_surprise_pct`) has dramatically stronger feature signal than the YoY-level target tested in early EDA — full-sample R²=0.81 vs 0.51.
2. In-sample feature signal exists but doesn't fully survive walk-forward at n≈21. Direction is robust; magnitude has wide CI.
3. β1 ≈ +2.59, not 0.9–0.95: DASH revenue overshoots GOV by ~2.6× via take-rate expansion + ads.
4. The naive single-stage β2 (revenue → EBITDA) is weak. Decomposing into pillar-aligned regressions reveals β_C (contrib → EBITDA) = +3.0 is the strong link; β_U (revenue → contrib margin) is weak.
5. Cross-sectional spread vs UBER is the sharpest L/S signal — direction set by the cross-sectional pillar, not stock-reaction β3.
6. β3 is weak (R²=0.02): DASH stock reactions are dominated by guidance / EBITDA / macro, not GOV surprise alone. Don't lean on β3 for magnitude.

## Disclaimers

- **n ≈ 21 quarters**. Every β coefficient is interpreted directionally,
  not as a precise point estimate. See the private spec §4 for the sample-size
  framework and §17 for empirical findings.
- **DASH-only models** for the GOV forecast (no pooling with UBER/CART per
  spec). UBER cross-sectional model trained on UBER data alone.
- **Look-ahead bias guards**: `validate_no_lookahead()` runs before every
  model fit. Trends/AppStore use 8-week mean ending 2 weeks before
  quarter-end. FRED uses 30-day-pre-quarter-end last value. IBES is
  snapped 60 days before quarter-end. Causal expanding-std and
  causal-trailing-mean for standardized / demeaned target variants.
- **CRSP coverage**: legacy DSI ends 2024-12-31; CIZ stock data extends
  to 2025-12-31. Q4 2024–Q3 2025 events use CRSP CIZ stock + SPY market
  proxy. Q4 2025 event (2026-02-18) uses full yfinance fallback.
- **Pre-registration**: `outputs/tables/q1_2026_preregistered.csv` was
  written + git-committed before the May 6 2026 print. Comparing
  realized surprise to the pre-registered point + CI is the out-of-sample
  test of the model.
